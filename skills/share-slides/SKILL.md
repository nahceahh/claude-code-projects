# share-slides

Create a shareable GitHub Pages link for an existing HTML slide deck (e.g. generated by the `frontend-slides` skill).

## When to use this skill
Use when the user has a local slide deck (HTML/CSS/JS) and wants a **public, shareable link** that others can open in a browser.

## Trigger
TRIGGER when the user asks to share slides, publish HTML slides, or create a public link for a local slide deck.
DO NOT TRIGGER when the user only wants to send files as a zip or is asking generally what GitHub Pages is.

## Assumptions (hard-coded defaults)
- Slide directory: `~/Desktop/skill-slides/`
- Entry file: `index.html`
- Host: **GitHub Pages**
- The result is **public** unless the user explicitly asks otherwise.

## Required tools
- Bash (for git / gh / filesystem checks)

## Safety / constraints
- Never guess or fabricate URLs. Only return URLs that come from `gh` output or the GitHub API.
- Ask before doing any operation that changes remote state (creating a repo, pushing commits).
- Do not commit secrets. If suspicious files exist (e.g. `.env`), stop and ask.

## Workflow
### 1) Verify local slide deck
1. Confirm `~/Desktop/skill-slides/index.html` exists.
2. Confirm required files exist (at minimum `index.html`; optionally `styles.css`, `slides.js`, `assets/`).
3. List directory contents so the user can confirm what will be published.

### 2) Confirm GitHub context
Ask the user:
- Do you want to publish under an existing repo or create a new repo?
- If new repo: desired repo name (default: `skill-slides`)
- GitHub owner/org (default: current authenticated `gh` user)

Verify `gh auth status` is logged in.

### 3) Publish to GitHub Pages
Support two paths:

#### Path A — Create a new repo (recommended default)
1. Create a new repo using `gh repo create <name> --public --source <local-dir> --remote origin --push`
   - If the CLI doesn’t support `--source` in this environment, initialize git in the directory, add remote, commit, and push manually.
2. Enable Pages:
   - Prefer `gh api` to configure Pages from `main` branch root.
   - If API fails, instruct user to enable via GitHub UI: Settings → Pages → Deploy from a branch → `main` / `/ (root)`.

#### Path B — Use an existing repo
1. Ensure the repo is cloned/initialized locally at `~/Desktop/skill-slides/`.
2. Commit slide artifacts to `main` (or user-chosen branch).
3. Push.
4. Enable Pages as above.

### 4) Return the shareable link
- Fetch the Pages URL using GitHub API (preferred) and print it.
- If Pages is still provisioning, explain it can take a minute and show where to find it in Settings → Pages.

## Output format
Return:
- Repo: `<owner>/<repo>`
- Pages URL: `https://...`
- What was published: list of files
- Notes: any manual steps required
